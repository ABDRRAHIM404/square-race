# 🟥 Maze Runners

A full-screen **maze** game where **4 colored squares bounce through twisting corridors like billiard balls**. Pick your square, then watch them ricochet off every wall, grab knives and shields scattered on the floor, and battle to reach the exit across **5 hand-built maze stages**. Highest cumulative score takes the podium.

Built with **pure HTML5 Canvas + Vanilla JavaScript** — no game engines, no image files, everything drawn programmatically. Wrapped to a native Android app with **Capacitor**, and an Android **debug APK is built automatically in the cloud** by GitHub Actions on every push to `main`.

---

## ✨ Features

- 🟨🟦🟩🟥 Four racer squares with distinct speeds (yellow fastest → blue slowest)
- 🧩 **5 hand-designed mazes** that fill the whole screen — open rooms, concentric rings, staircases, parallel teeth, scattered pillars
- 🎱 **Billiard physics** — squares move in straight lines and reflect off every wall (outer boundary + internal maze walls), fully passive, no steering
- ⚔️ **Loot on the floor**: grab a **knife** to eliminate squares you touch, or a **shield** to absorb one knife hit (one item at a time)
- 🏁 Reach the exit to bank your placement; squares hit by a knife-carrier spin out and are placed last
- 🏆 Points each stage (1st=4, 2nd=3, 3rd=2, 4th=1) → a **final podium ranked by cumulative score** after 5 stages
- 🔊 Web Audio sound effects (no audio files), with a mute button
- 📱 Mobile-first, portrait, full-screen, works offline after install

## 📦 Get the Android APK (the easy way — no local setup)

The repo ships with a GitHub Actions workflow that builds the APK for you in the cloud.

1. **Create a new GitHub repo** (e.g. `maze-runners`).
2. **Push this project to it:**
   ```bash
   cd square-race
   git init
   git add .
   git commit -m "Maze Runners"
   git branch -M main
   git remote add origin https://github.com/<your-username>/maze-runners.git
   git push -u origin main
   ```
3. On GitHub, open the **Actions** tab. The **"Build Android APK"** workflow runs automatically on the push (you can also trigger it manually via **Run workflow**).
4. When the run finishes (green check), open it and scroll to **Artifacts**. Download the APK artifact — it contains `app-debug.apk`.

> The `android/` folder is intentionally **not** committed (it's in `.gitignore`). The CI regenerates it with `npx cap add android` on every build, so the repo stays clean.

---

## 📲 Install the APK on your phone

1. Copy `app-debug.apk` to your Android device (USB, email, cloud drive…).
2. In **Settings → Security**, enable **Install unknown apps** for the app you're opening it from (e.g. your browser or file manager). On newer Android you'll be prompted automatically.
3. Tap the APK and install. Launch **Maze Runners**.

> This is an unsigned **debug** build — perfect for personal use and sideloading. It is not meant for the Play Store.

## 🛠️ Build the APK locally (optional)

Requires **Node.js 20+**, **JDK 17**, and the **Android SDK** (Android Studio is the easiest way to get it).

```bash
cd square-race
npm install
npx cap add android       # generates the native android/ project
npx cap sync android      # copies www/ into the app
cd android
./gradlew assembleDebug   # on Windows: gradlew.bat assembleDebug
```

The APK lands at:

```
android/app/build/outputs/apk/debug/app-debug.apk
```

Whenever you change anything in `www/`, run `npx cap sync android` again before rebuilding.

## 🎯 How it works

| Screen        | Action                                                        |
|---------------|---------------------------------------------------------------|
| Title         | Tap anywhere to start                                         |
| Select        | Tap a colored square, then tap **START**                      |
| Stage Intro   | Brief "STAGE N" countdown before the maze begins             |
| Race          | Watch the squares bounce; tap 🔊 to mute                      |
| Stage Result  | See placements + points, then tap **NEXT STAGE**             |
| Final Result  | Podium by total points — **Play Again** or **Change Square**  |

The squares play themselves — your choice of color (and a bit of luck with the loot) decides how they fare. **Knife** beats an unshielded square on contact; a **shield** soaks one knife hit then breaks.

## ⚙️ Tech notes

- **No external libraries** — pure Canvas 2D API.
- **No image assets** — every visual is drawn in code.
- **Deterministic stages** — a `mulberry32` seeded PRNG drives loot placement and launch directions.
- **Frame-rate independent** — all movement uses `deltaTime`.
- **Guaranteed termination** — a gentle exit-attraction bias plus a hard per-stage time cap ensure every stage ends, even though the squares bounce passively.
- **Target:** Android 7.0+ (API 24+), portrait, offline-capable.
